BP Agro

Plataforma comercial de monitoreo IoT de suelos para fincas agrícolas. Sensores de campo Raspberry Pi 3, gateway por finca con gestión de zonas y cultivos, almacenamiento en AWS Aurora y alertas en tiempo real al agricultor via SMS y WhatsApp.

Nodo de Campo
bp-agro-sensor-node
Raspberry Pi 3 instalado en cada zona de la finca. Lee sensores de suelo via RS-485 bus cada 15 minutos: humedad, pH, temperatura, NPK y conductividad eléctrica. Valida rangos localmente y envía lecturas al gateway por HTTP LAN. Sin internet — solo red local.
PythonRPi 3 RS-485GPIONTP sync

Gateway de Finca
bp-agro-farm-gateway
Pi 4 o x86 por finca. Agrega lecturas de todos los sensores, gestiona zonas y cultivos (50 tipos, historial permanente), y sube datos a Aurora con pre-verificación de duplicados por machine_id + timestamp. Dual-path: sync queue (fiable) + notify inmediato (rápido). Zone rename, bulk crop assignment, calibración.
PythonFlask SQLiteAPScheduler Aurora uploadZone Manager

Decisión de diseño
Pre-existence Check vs INSERT IGNORE
¿Por qué no INSERT IGNORE?
La clave primaria AUTO_INCREMENT hace INSERT IGNORE incorrecto para deduplicación
El campo id en Aurora usa AUTO_INCREMENT como identificador FCFS (First-Come-First-Served). Esto significa que cada intento de INSERT genera un nuevo ID aunque sea un duplicado de machine_id + timestamp — INSERT IGNORE simplemente descartaría la fila sin verificar nada útil. La solución correcta es una pre-verificación explícita: antes del batch upload, el gateway consulta Aurora por las combinaciones machine_id + timestamp del batch completo, filtra los ya existentes, y solo inserta registros genuinamente nuevos. Todos los históricos se conservan permanentemente — el botánico los necesita completos.
📊 Flujo: Sensor → Gateway → Aurora (con pre-check)
Ciclo completo de 15 min + batch upload
Stack tecnológico
Tecnología Utilizada
🔬
RS-485 + GPIO
Bus de sensores industriales. Humedad, pH, temperatura, NPK, conductividad eléctrica del suelo.
🐍
Python + Flask
Gateway y Business API. APScheduler para ciclos de upload. SQLite local en el gateway.
☁️
AWS Aurora MySQL
BD maestra cloud. Todos los históricos permanentes. Fuente para alertas y analytics.
📲
Africa's Talking + WhatsApp
Notificaciones de alerta al agricultor. SMS para zonas sin internet. WhatsApp como canal principal.
🖥️
Raspberry Pi 3 / 4 / x86
Pi 3: nodo sensor de campo. Pi 4 / mini-PC: gateway por finca según tamaño de explotación.
📱
App QR (asignación nodos)
El técnico escanea el QR del Pi 3 para asignarlo a una zona del gateway. Sin pasar por la Business API.
