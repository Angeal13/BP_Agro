A — Nodo Sensor (Raspberry Pi 3) — bp-agro-sensor-node
El Pi 3 lee sensores de suelo via RS-485 o GPIO cada 15 minutos. Valida rangos localmente y envía lecturas al gateway por HTTP LAN. Sin conexión a internet — solo red local de la finca. Humedad · pH · temperatura · NPK · conductividad eléctrica.
Lecturas activas
PLATAFORMA COMERCIAL
B — Gateway de Finca (Pi 4 / x86) — bp-agro-farm-gateway
El gateway agrega lecturas de todos los sensores de la finca, gestiona zonas y cultivos (50 tipos, historial de rotación permanente), y sube datos a AWS Aurora con pre-verificación de duplicados. Dos paths de propagación: sync queue (fiable) y notify inmediato (rápido).
Eventos activos
C — Pipeline Cloud: Aurora → API → Alertas
AWS Aurora recibe todos los datos de las fincas. La Business API gestiona clientes y fincas (sin rutas de gateway o sensor — separación explícita). El motor de alertas polling Aurora detecta umbrales críticos y notifica al agricultor por SMS/WhatsApp.
Eventos activos
D — Dashboard Botánico — bp-agro-botanist-dashboard
Dashboard HTML con fetch() a la API real — sin mocks ni datos simulados. El botánico o agrónomo accede al historial completo sin procesar por zona y cultivo. Datos de calibración, rotación de cultivos y tendencias temporales de todos los parámetros.
Eventos activos
E — Resumen General: Campo → Gateway → Aurora → Usuarios
Vista completa del sistema: sensores leyendo el suelo, gateways agregando y subiendo a Aurora, motor de alertas notificando al agricultor, botánicos analizando datos históricos y la API de negocio gestionando clientes.
